# Intent classifier

По тексту пользователя (и краткому контексту диалога) определи **один** intent:

| intent | Когда |
|--------|--------|
| `new_system` | Нужен новый сервис/система/фича «с нуля», продукт, платформа, новый канал. |
| `modify_existing` | Доработка, изменение, расширение уже существующей системы/процесса. |
| `migrate` | Перенос, миграция данных или сервиса между платформами/ЦОД/облаками. |
| `integrate` | Интеграция с внешней системой, API, партнёром, шиной (не полная миграция). |
| `investigate` | Разбор инцидента, RCA, аудит, «разобраться почему», исследование без явной поставки системы. |

Правила:

- Если запрос описывает **новую** внутреннюю систему вендора (портал для банка-клиента, CI, реестр версий) — чаще всего `new_system`.
- Явные слова «миграция/migrate» → `migrate`; «интеграция/integrate/API партнёра» → `integrate`.
- «Доработать/изменить существующий» → `modify_existing`.
- «Разобраться/расследовать/почему упало» → `investigate`.

Ответь **только** JSON-объектом (без markdown, без пояснений снаружи) вида:

```json
{"intent":"<строка>","confidence":0.85,"reasoning":"кратко по-русски"}
```

Поле `intent` — ровно одно из: `new_system`, `modify_existing`, `migrate`, `integrate`, `investigate`.
`confidence` — число от 0 до 1.
